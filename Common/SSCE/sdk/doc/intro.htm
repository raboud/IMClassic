Document Overview

This Programmer's Guide describes how to use the Sentry Spelling-Checker Engine (SSCE) with a software application. This document applies to the Sentry Spelling Checker Engine Windows SDK (Software Development Kit) and the Sentry Spelling Checker Engine Source SDK. Information that is specific to one SDK is clearly indicated as such; otherwise, information applies to both SDKs.

Sentry Software Development Kit Overview

The Sentry Spelling Checker Engine is a library you can use to add a spelling-checker capability to your applications. Sentry is intended for software developers and application builders. Sentry is not a stand-alone spelling checker; it must be used within an application. The Sentry engine is available as 16- and 32-bit SDKs (Software Development Kits) for Microsoft Windows, a portable C-language Source SDK, a Java class library, and Java source code. All SDK types include an API (Application Program Interface) you can use to check the spelling of words, add words to user dictionaries, get suggested replacements for misspelled words, etc. The Windows SDKs include built-in spelling checker dialog boxes, and have an augmented API that includes functions specific to the Windows environment.

This programmer's guide applies to the Sentry Windows SDK and the C Source SDK. Each SDK includes:

  • The Sentry Spelling-Checker Engine implemented as either 16/32 bit Windows DLLs (Windows SDK) or ANSI C source code (Source SDK)

  • An American English main lexicon containing about 100,000 words

  • A British English main lexicon containing about 100,000 words

  • An auto-correction lexicon containing hundreds of common English misspellings and their correct replacements

  • This programmer's guide

  • Source code for applications which demonstrate how to use the Sentry engine

  • A utility program used to create compressed lexicons from word lists.

Sentry Spelling Checker Engine Capabilities

Sentry provides a general-purpose spelling-checker capability for software applications. You can use Sentry from your application to check the spelling of words (e.g., in a word-processor application) or to classify text strings as words (e.g., in an OCR or handwriting-recognition application).

Using the Sentry spelling-checker engine is a co-operative venture between your application and Sentry's API.

Sentry provides:

  • Fast word lookups in any number of lexicons (dictionaries)

  • Highly compressed lexicons

  • Support for multi-lingual lexicons with words formed from the Latin 1 (ISO-8859-1) character set. Through a special build option, the Source SDK can support the Unicode character set.

  • Checking for uncapitalized words, words with unusual case combinations, doubled words, and words containing embedded digits

  • Checking of individual words or blocks of text

  • Intelligent suggested replacements for misspelled words using phonetic and typographical matching

  • Support for ignore-all and replace-all word processing

  • Ability to automatically correct common misspellings

  • Ability to automatically suggest specific replacements for certain words

  • Ability to mark words as misspelled, even if they exist in other lexicons

  • Support for text-lexicon maintenance (adding and removing words)

  • Support for undoing corrections made to text

  • Ability to automatically expand abbreviations

  • Ability to create compressed lexicons from word lists.

Your application provides:

  • All user interface components (however, a user interface for Microsoft Windows applications is provided with the Windows SDK)

  • Extraction of words from a document

  • Replacement of corrected words in a document

  • General co-ordination of the spell-checking process.

Glossary

  • Block: A block is a character array (string) containing one or more words. Sentry's Core API contains functions for checking the spelling of and modifying words in blocks.

  • Character set: A mechanism for mapping character shapes to numeric values. The Sentry engine supports the Latin 1 character set, and can support the Unicode character set through a special build option (Sentry Source SDK only).

  • Compressed lexicon: A compressed lexicon is a set of words in compressed form. Like a text lexicon, a compressed lexicon contains words, but the words are stored in a special form that eliminates redundancies so the words occupy less space. On average, a compressed lexicon is slower to access than a text lexicon. Words cannot be added to or removed from a compressed lexicon. Compressed lexicons can hold hundreds of thousands of words. See also Lexicon and Text lexicon.

  • Latin 1: A standard method of mapping character shapes to single-byte numerical values. Characters with values ranging from 0 to 127 are identical to the ASCII character set. Characters with codes above 127 are used to represent accented characters. Latin 1, also known as ISO-8859-1, is the same as the ANSI character set used in Microsoft Windows and other operating systems.

  • Lexicon: A lexicon is a set of words. Some spelling checkers use the term "dictionary" to describe the set of words they use.

  • Lexicon format: The organization and structure of a lexicon. Sentry supports two lexicon formats: text, which hold at most a few thousand words and are modifiable, and compressed, which can hold an unlimited number of words but are not modifiable.

  • Main lexicon: The main lexicon is a large collection of commonly used words in a specific language. The main lexicon is normally compressed and is not modifiable by users at run-time.

  • Permanent lexicon: Words in a permanent lexicon are saved to disk when the lexicon is updated. When the lexicon is reopened, its words will be restored from its disk image. Compressed lexicons are always permanent; text lexicons may be permanent or temporary. See also Temporary lexicon.

  • Suggestion: The Sentry engine can suggest alternative words which are likely replacements for a misspelled word. For example, if the misspelled word is stopp, Sentry might suggest stop, steppe, stopped, etc. Sentry will locate the n best alternative replacements for a word; your application determines how large n is.

  • Temporary lexicon: Words in a temporary lexicon exist only while the lexicon is open. Once the lexicon is closed, the words are lost. Only text lexicons may be temporary. See also Permanent lexicon.

  • Text lexicon: A text lexicon is a set of words in text form. Words can be added to or removed from a text lexicon. Text lexicons are generally fast to access, but they take up more space per word than compressed lexicons. Text lexicons are sometimes called "user lexicons" or "user dictionaries." See also Compressed lexicon.

  • Unicode: A standard way of mapping character shapes to double-byte numerical values. Unicode can represent character shapes in virtually all human languages.

  • User lexicon: See Text lexicon.
